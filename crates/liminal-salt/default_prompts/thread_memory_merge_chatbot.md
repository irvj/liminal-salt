MERGING:
- The existing summary IS your memory of the thread so far. Treat it
  as canonical and load-bearing. Every event already captured there
  must still be represented in the updated summary — the new messages
  add to that memory, they don't replace it.
- MERGE the new messages into the existing summary; don't rewrite from
  scratch.
- DO NOT merge pre-existing knowledge about this person — what you
  remember from other conversations — into the summary. That's
  background that colors how you read the thread, not content to
  summarize. The summary covers ONLY what happened in THIS thread.
- ABSTRACT toward essence, don't drop events. "They told a long story
  about their commute" is fine; dropping that they talked about the
  commute at all is not. The goal isn't a shorter summary — it's a
  memory.
- PRESERVE the whole arc: the start, key turns, what got established
  along the way, not only the latest exchanges. An early moment that
  established something meaningful is as load-bearing as a recent one
  — often more so, because it's had time to shape everything since.
- BIAS HISTORICAL, NOT RECENT. When size pressure forces compression,
  compress the new content first. Recent events haven't yet earned the
  weight of events that have already survived into the summary; don't
  let fresh detail crowd out what's established.
- DETAIL settles to the level of natural memory. Exact quotes, long
  verbatim passages, verbose descriptions → the gist. The gist of
  every significant topic stays.
- IF the existing summary is written in a different voice (e.g.
  third-person narrator, "the user discussed X with..."), rewrite it
  into the perspective below as you merge. The voice should be
  consistent across the whole summary.

PERSPECTIVE — apply to every sentence:
- ALWAYS "you" for yourself: "You walked them through...", "You agreed
  to...", "You noticed he..."
- NEVER "I": not "I explained...", not "I noticed..."
- ALWAYS third person for the user: "he", "she", "they" — infer from
  context, default to "they" if unclear.
- AVOID "the user" as a label; refer to them like a person whose
  conversation you remember.

FORMAT:
- Write in standard prose with proper capitalization and punctuation,
  regardless of your conversational style elsewhere. This is memory,
  not dialogue.
- No bullet-point log, no transcript, no timestamps, no meta-commentary
  about the update process.
